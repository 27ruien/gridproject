# Kiviflow 体验与视觉重构规范

## 1. 信息架构

全局导航：
- 工作台：项目经理从单项目经营角度查看工时、进度、待办、完结、风险和下一步跟进。
- 项目库：所有项目的列表、健康度、进度和模板。
- 工时填报：创建、查看和编辑关联项目/任务的工时记录，并用视角模拟权限隔离。
- 创建项目：模板选择只在创建流程中出现，避免无自定义能力的模板中心占据导航。

项目空间：
- 顶部摘要：项目名称、模板、负责人、进度、风险、下一步。
- 视图切换：由模板决定默认视图集合。
- 主工作区：敏捷展示看板/Backlog/Sprint；瀑布展示阶段/里程碑/交付物/风险。
- 右侧详情抽屉：查看和编辑事项。

## 2. 创建项目流程

- 从工作台或项目库进入创建弹窗。
- 左侧填写项目名称、负责人、起止日期、说明。
- 右侧选择模板，并展示该模板会初始化的视图、事项类型和空状态引导。
- 创建后直接进入项目空间，并给出成功反馈。

## 3. 关键页面

工作台 / 项目列表：
- 需要展示项目总数、未完成事项、高风险事项、平均健康度。
- 最近项目以列表呈现，便于扫描负责人、模板、进度、风险。
- “我的工作”展示跨项目事项，可快速推进状态。

敏捷看板：
- 四列状态：未开始、进行中、已完成、已验收。
- 卡片显示编码、类型、优先级、负责人、截止日期和下一步。
- 支持从卡片直接推进状态。

甘特图：
- 以任务开始日期和截止日期绘制排期条。
- 支持点击任务打开详情。
- 与事项列表和看板共用日期、关键词、执行人、创建人筛选条件。

瀑布阶段视图：
- 以阶段门方式展示立项、需求确认、设计确认、开发实施、测试验收、交付归档。
- 每阶段显示关联事项数量、风险提示和交付物。

事项列表：
- 表格密度适中，列为类型、标题、负责人、优先级、截止日期、状态。
- 支持点击进入详情抽屉。

事项详情：
- 必须能编辑标题、类型、负责人、优先级、截止日期、状态、描述、下一步。
- 展示所属项目、模板和活动记录。

## 4. 状态体验

- 空状态：给出当前视图应做的下一步，不使用空白页面。
- 加载状态：当前纯本地应用加载很快，保留样式钩子。
- 错误状态：表单必填缺失以 Toast 反馈。
- 操作反馈：创建、保存、推进状态均出现 Toast。

## 5. 商业化 SaaS 视觉方向

- 克制、清晰、可扫描，偏成熟工作台而非营销页面。
- 色彩以中性背景、白底导航、蓝色品牌色、蓝/绿/橙/红语义色组成，避免花哨渐变。
- 卡片半径控制在 8px，优先用于项目、事项、模板等独立对象。
- 信息密度接近项目管理工具，桌面端优先，移动端基础适配。

## 6. 组件规范

- 按钮：主按钮用于创建/保存，次按钮用于查看/取消，小按钮用于行内操作。
- 输入框：统一 40px 高度，清晰边框和聚焦态。
- 选择器：用于模板、类型、负责人、优先级、状态。
- 状态标签：未开始灰蓝、进行中蓝、已完成绿、已验收深绿。
- 优先级标签：P0 红、P1 橙、P2 蓝、P3 灰。
- 卡片：用于模板、项目、看板事项。
- 表格：用于事项列表和工作台事项。
- 看板列：固定最小宽度，横向滚动，列头显示数量。
- 详情面板：右侧抽屉，支持编辑与活动记录。
- 顶部工具栏：搜索、创建项目、创建事项、视图切换。
