# GridProject Dev Known Issues

测试版本：`0.1.0-dev.1`

更新日期：`2026-06-21`

## 尚未最终确认的产品权限

- 当前后端已限制组织边界、工时可见范围、成本管理权限、项目成员管理权限和事项删除权限。
- 项目列表和项目看板仍按当前组织内可见处理，是否进一步限制为“仅项目成员可见”需要产品最终确认。
- 事项编辑权限目前宽于删除权限，删除仅允许 ADMIN、项目 Owner 或事项创建者。是否将事项编辑也收紧到相同角色需要产品确认。
- 成本管理当前仅允许 ADMIN 和项目 Owner 访问，普通 MEMBER 不应测试成本录入或导出。

## localStorage 模式与 API 模式差异

- `localStorage` 模式用于前端演示和静态交互，不具备后端登录、会话、组织隔离、审计日志和数据库持久化能力。
- API 模式通过 `VITE_DATA_SOURCE=api` 读取 Fastify 后端，数据写入 PostgreSQL，并启用权限校验和 AuditLog。
- Dev UAT 只应测试 API 模式；如发现 localStorage 模式行为差异，不作为本轮 Dev RC 阻断项。

## 不建议测试的功能

- 不建议测试生产数据导入、真实财务金额、真实人员成本、真实合同信息。
- 不建议用真实客户项目或真实员工账号进行测试。
- 不建议压测、破坏性批量删除、绕过页面直接改库。
- 不建议测试 Prod Nginx、Prod systemd、Prod 数据库或 Prod 账号。

## Dev 数据规则

- Dev 数据可能会在测试结束后清理。
- Dev 数据不得录入真实敏感信息。
- 只允许使用虚拟人员姓名、测试邮箱、测试项目、测试工时和非敏感描述。
- 禁止录入真实工资、真实人员成本、客户敏感数据、生产账号密码、生产项目数据、真实合同或财务信息。
