# Kiviflow 产品化重构验收报告

## 1. 重构完成摘要

已将 Kiviflow 从 Vue 3 + Vite 单文件 MVP 重构为具备产品文档、设计规范、领域模型、服务层、存储适配器、组合式状态、组件化视图和轻量测试的项目管理平台前端雏形。

## 2. 产品层变化

- 明确目标用户：研发项目负责人、交付 PM、团队成员和管理者。
- 明确核心对象：Workspace、Project、Template、Issue、Workflow、View、Milestone、Activity。
- 明确模板不是切页面，而是决定默认视图、事项类型、字段、工作流、空状态和初始化事项。
- 明确第一版不做后端、登录、权限、飞书和复杂报表。

## 3. 设计层变化

- 首屏改为可用工作台，不做营销页。
- 建立白底左侧导航、顶部搜索、工作台、项目库、工时填报、项目工作现场、详情抽屉。
- 敏捷项目突出 Backlog、Sprint、看板。
- 瀑布项目突出阶段计划、里程碑、交付物、风险、验收。
- 组件规范覆盖按钮、输入、选择器、标签、项目行、事项表格、看板列和详情面板。

## 4. 技术架构变化

- `src/domain/`：领域模型与工作流规则。
- `src/storage/`：storage adapter 和 localStorage adapter。
- `src/services/`：项目、事项、模板、状态服务。
- `src/composables/`：项目、事项、项目空间状态组合。
- `src/components/`：通用、模板、项目、事项组件。
- `src/views/`：Dashboard、项目创建、项目空间视图。
- `src/router/routes.js`：前端导航配置，为未来真实路由预留入口。

## 5. 已实现功能

- 专业 Dashboard / 工作台。
- 项目经理工作台：项目视图和待办事项作为同级模块展示。
- 项目视图卡片弱化敏捷/瀑布模板标签，突出项目阶段、健康度、剩余工时、截止风险、进度和关键任务指标。
- 待办事项支持“我的 / 即将到期 / 风险逾期”Tab，帮助项目经理安排当天动作。
- 已移除重复的项目组合信号模块。
- 工作台去除创建项目、查看项目库等分心按钮，聚焦项目经理日常经营判断。
- 项目库、工时填报；去除当前无闭环的运营报表入口。
- 创建项目流程，支持敏捷研发和瀑布交付模板。
- 模板驱动项目初始化和种子事项。
- 项目空间摘要：项目名称只出现一次，项目状态、健康度、进度、待办、投入和关键日期按主次分层展示。
- 敏捷看板、瀑布阶段视图、甘特图、事项列表。
- 事项创建、编辑、详情抽屉、状态流转。
- 任务筛选支持日期范围、关键词、执行人和创建人。
- 已下线无闭环的顶层模板中心；模板只保留在创建项目流程中。
- 新增平台设置，支持维护左上角 Logo 文案和平台名称，去除写死的“项目管理平台”和“Kivisense 组织工作区”文案。
- 任务详情支持评论，人员选择改为可搜索弹层。
- 任务详情抽屉增加顶部 Tab，评论和工时不再混在长表单底部。
- 新增解耦的工时填报数据模型，工时记录关联项目和任务，支持创建、查看、编辑和单次多任务申报。
- 工时填报模拟权限隔离：我负责的查看自己管理项目的成员填报，我提交的查看自己的跨项目填报。
- 工时列表默认展示当月，支持月份筛选、全字段搜索、项目、任务、填报人、日期和工时范围筛选。
- 我提交的展示本月应申报工时、当前已申报工时和待补日期，创建申报时自动定位到待补日期。
- 项目页新增测试时间、验收时间和上线时间展示，并支持项目经理手动修改项目当前状态。
- 项目支持编辑和软删除；项目下仍有任务时禁止删除，并提示“项目下还有 N 个任务。请先删除或迁移任务，再删除项目。”
- 任务支持从详情抽屉软删除。
- 新增回收站，作为项目库内的数据管理入口，30 天内可恢复删除的项目或任务。
- 项目页抬头信息合并为经营信号面板，项目状态改为胶囊按钮组，弱化孤立下拉操作。
- 项目概览新增风险雷达和最近动态，项目经理可以先看需要处理的风险，再追踪本项目事实流。
- 项目级里程碑由模板初始化并持久化，支持在项目概览和里程碑视图维护阶段状态，项目顶部展示里程碑进度。
- 侧栏项目快捷入口已移除。
- localStorage 持久化与旧数据归一化。
- 轻量领域逻辑测试。

## 6. 未做功能和原因

- 后端、登录、权限、飞书：用户明确第一版不做。
- 拖拽看板：避免引入复杂依赖，当前用状态选择与推进按钮满足闭环。
- 模板编辑器：当前聚焦两套商业化模板雏形，先保留模型扩展点。
- 运营报表：当前阶段缺少可操作闭环，已移除入口；后续应在真实多项目经营指标、导出和权限明确后再做。

## 7. Jira / Teambition 对标结论

当前达到商业化前端雏形，但不是 Jira 或 Teambition 的完整替代。与 Jira 相比，仍缺复杂工作流、权限、自动化和生态；与 Teambition 相比，仍缺协作、日历、文件、评论和通知。当前优势是轻量、清晰、模板差异明确，适合继续产品化。

## 8. 验证命令和结果

- `npm install`：成功。
- `npm run test`：成功，输出 `domain tests passed`。
- `npm run build`：成功。
- `npm run dev`：成功，本地服务为 `http://127.0.0.1:5173/`。
- `curl -I http://127.0.0.1:5173/`：返回 `HTTP/1.1 200 OK`。

## 9. 剩余风险

- 浏览器自动化插件连接超时，未能完成截图级验收；已通过构建、测试和本地 HTTP 可访问性验证。
- localStorage 仍是单机存储，未来接后端时需要实现 API adapter。
- 当前无真实并发协作，活动记录为本地事件。

## 10. 验收者重点检查

- 工作台：是否已去掉无意义的长条口号模块，首屏直接进入指标、项目视图和我的待办事项。
- 工作台：项目视图和待办事项是否为同级模块，项目组合信号是否已移除。
- 工作台：项目卡片是否弱化敏捷/瀑布标签、突出项目阶段，并将进度百分比和进度条合并展示。
- 工作台：待办事项 Tab 是否能区分我的、即将到期、风险逾期。
- 项目页：项目当前状态是否可以由项目经理手动修改。
- 项目页：项目基础信息是否可以编辑；有任务的项目删除时是否出现清理任务提示。
- 任务详情：删除任务后是否进入回收站。
- 回收站：30 天内删除的项目或任务是否可以恢复。
- 工时填报：我负责的和我提交的是否展示不同数据范围，是否能创建和编辑关联项目/任务的工时。
- 工时填报：单次创建是否能添加多条任务明细；月份筛选和全字段筛选是否生效；待补日期是否能引导创建。
- 项目页：顶部是否只出现一次项目名称，健康度/进度是否比待办/工时/关键日期更突出。
- 项目页：是否展示测试时间、验收时间和上线时间。
- 项目概览：风险雷达是否能优先暴露逾期、P0、风险和临近到期事项；最近动态是否能打开对应事项。
- 里程碑：新建项目是否自动生成里程碑；修改里程碑状态后刷新页面是否保留。
- 项目库：回收站是否只作为项目库内入口，不再占据左侧一级菜单。
- 平台设置：修改 Logo 文案和平台名称后，左上角品牌是否立即更新。
- 任务详情：能否添加评论，能否用弹层选择执行人/创建人。
- 创建项目：分别创建敏捷研发和瀑布交付项目。
- 项目空间：敏捷看板、瀑布阶段/里程碑是否有明显差异。
- 事项流程：创建事项、打开详情、编辑字段、推进状态。
- 甘特图：查看不同任务的起止日期和排期跨度。
- 任务筛选：用日期、关键词、执行人、创建人组合查询事项。
- 持久化：刷新页面后项目和事项是否仍存在。
