# Kiviflow 体验测试与对标审计

## 测试场景

- 新用户进入产品后是否知道下一步做什么：通过。工作台首屏直接提供跨项目指标、项目视图和待办事项，不再混入创建项目与模板浏览按钮。
- 创建敏捷研发项目：通过。模板选择会初始化敏捷视图和种子事项。
- 创建瀑布交付项目：通过。模板选择会初始化瀑布视图和种子事项。
- 创建事项：通过。事项包含类型、标题、负责人、优先级、截止日期、描述、下一步。
- 编辑事项：通过。右侧详情抽屉可编辑关键字段并保存。
- 推进事项状态：通过。看板、列表、详情抽屉均可推进。
- 查看项目进度：通过。项目空间顶部展示健康度、进度、待办、投入工时和测试/验收/上线时间。
- 判断当前工作重点：通过。项目概览用风险雷达优先展示逾期、P0、风险和临近到期事项。
- 从空项目建立真实数据：基本通过。模板空状态提供新建引导。
- 切换模板后的体验差异：通过。敏捷有 Backlog/Sprint/看板，瀑布有阶段/里程碑/交付物/风险/验收，且项目里程碑会从模板初始化为可维护状态。

## P0/P1/P2/P3 问题

P0：无。

P1：
- 旧版 localStorage 项目缺少起止日期会影响项目库扫描。已修复：`stateService` 增加项目归一化。

P2：
- 暂未支持拖拽看板，只支持选择状态和推进按钮。
- 里程碑已支持项目级持久化和状态维护，后续可补充负责人、依赖关系和交付物附件。
- 缺少真正的权限、成员和通知机制。

P3：
- 移动端为基础适配，复杂项目管理操作仍建议桌面端。
- 报表仍是轻量概览，未支持自定义维度和导出。

## 与 Jira 的差距

- Jira 强在复杂工作流、自定义字段、权限、自动化、报表和生态集成；Kiviflow 当前只完成前端雏形与模板化体验。
- Kiviflow 当前优势是启动更轻、模板语义更直观、适合商业化原型验证。
- 后续要接近 Jira，需要补后端、权限、可配置工作流、拖拽看板、筛选器、活动通知和高级报表。

## 与 Teambition 的差距

- Teambition 强在项目协作、日历、文件、动态和团队空间；Kiviflow 已补充本地项目动态流，但仍没有多人协作与文件域。
- Kiviflow 当前更强调敏捷/瀑布模板差异和项目空间的下一步判断。
- 后续要接近 Teambition，需要补成员协作、附件、通知、日历视图和团队空间管理。

## 当前是否达到商业化雏形

达到。它已经从单文件 demo 变成有产品定位、模板驱动模型、商业化 SaaS 界面、可运行核心流程和可替换存储边界的前端雏形。

## 必须修复的问题

无 P0。已修复 P1 旧数据迁移显示问题。
